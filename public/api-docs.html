📚 NBBB API 接口文档

NBBB 电商系统 API 接口完整文档

📋 接口概览

本文档描述了 NBBB 电商系统的所有 API 接口，包括认证、商品、订单、支付、购物车等功能。

  • 服务器地址：https://not-boringboreboi.com
  • API 前缀：/api
  • 认证方式：Bearer Token（大部分接口需要）
  • 接口总数：约 68 个接口

🔐 认证说明

大部分接口需要在请求头中携带认证信息（登录接口除外）：

Authorization: Bearer <JWT_TOKEN> Content-Type: application/json

注意：登录接口（POST /api/auth/wechat/login）不需要认证，其他接口都需要在请求头中携带有效的 JWT Token。

1. 认证与登录接口

4. 购物车接口

⚠️ 特殊业务规则

  • 物品无法叠加：每个商品/SKU在购物车中只能有一条记录
  • 数量上限：每件衣服最多购买3件
  • 拆分展示：返回给小程序时，每个商品按数量拆分成多条独立记录，每条quantity=1

POST /api/cart/add

功能描述：添加商品到购物车。如果商品已存在，更新数量（不超过3件上限）
认证要求：需要 Bearer Token
参数名 类型 必填 说明
product_code String 必填 商品编码（来自聚水潭）
sku_id String 可选 SKU编码（如果商品有规格）
quantity Number 可选 数量（1-3之间，默认1）
请求示例：
Headers: Authorization: Bearer <JWT_TOKEN> Content-Type: application/json Body: { "product_code": "NBB-AWTR004", "sku_id": "NBB-AWTR004-1-L", "quantity": 1 }
响应示例（成功）：
{ "code": 0, "message": "添加成功", "data": { "id": 1, "product_code": "NBB-AWTR004", "sku_id": "NBB-AWTR004-1-L", "quantity": 1, "is_new": true, "is_max": false } }
响应示例（已达上限）：
{ "code": 400, "message": "该商品已达到购买上限（最多3件）", "data": { "current_quantity": 3, "max_quantity": 3 } }

GET /api/cart

功能描述：获取购物车列表。返回时会将每个商品按数量拆分成多条独立记录，每条quantity=1
认证要求：需要 Bearer Token
⚠️ 库存检查：每次请求时会自动检查所有商品的有效性和库存状态，无效商品会被标记为 valid: false，并包含 invalid_reason 字段说明原因。
响应示例：
{ "code": 0, "data": { "items": [ { "id": 1, "cart_item_id": 1, "product_code": "NBB-AWTR004", "sku_id": "NBB-AWTR004-1-L", "quantity": 1, "selected": true, "is_max": false, "original_quantity": 2, "index": 1, "created_at": "2025-11-14T...", "updated_at": "2025-11-14T..." }, { "id": 1, "cart_item_id": 1, "product_code": "NBB-AWTR004", "sku_id": "NBB-AWTR004-1-L", "quantity": 1, "selected": true, "is_max": false, "original_quantity": 2, "index": 2, "valid": true, "product": { "i_id": "NBB-AWTR004", "name": "商品名称", "pic": "https://...", "s_price": 1699 }, "created_at": "2025-11-14T...", "updated_at": "2025-11-14T..." }, { "id": 2, "cart_item_id": 2, "product_code": "NBB-INVALID", "sku_id": null, "quantity": 1, "selected": true, "is_max": false, "original_quantity": 1, "index": 1, "valid": false, "invalid_reason": "商品不存在", "product": null, "created_at": "2025-11-14T...", "updated_at": "2025-11-14T..." } ], "summary": { "total_items": 1, "total_quantity": 2, "total_expanded_items": 2, "selected_items": 1, "selected_quantity": 2, "valid_items": 1, "invalid_items": 0 } } }
字段 类型 说明
cart_item_id Number 购物车项ID（用于更新/删除操作）
quantity Number 固定为1，每条记录独立展示
original_quantity Number 原始数量（用于前端显示）
index Number 在同一条购物车项中的序号（1, 2, 3...）
is_max Boolean 原始商品是否达到上限（3件）
valid Boolean 商品是否有效（true=有效，false=无效）
invalid_reason String 无效原因（仅在valid=false时存在）：商品不存在/商品已下架/SKU不存在/SKU已下架
product Object 商品详细信息（从聚水潭获取，包含名称、价格、图片等）
无效商品处理：
  • 无效商品不会拆分，只返回一条记录
  • 无效商品包含 invalid_reason 字段说明原因
  • 前端可以根据 valid 字段显示不同的UI状态
  • 统计信息中包含 valid_items 和 invalid_items 字段

5. 订单接口

订单系统说明

  • 订单创建：从购物车创建订单，自动验证商品有效性
  • 订单存储：订单存储在本地数据库，包含完整的订单信息
  • 同步到聚水潭：订单创建后自动同步到聚水潭系统（异步执行），支付成功后也会自动同步
  • ID对应关系：维护本地订单ID和聚水潭订单ID的对应关系
  • 订单状态：支持订单状态查询和同步状态查询

POST /api/orders

功能描述：创建订单。从购物车中选择商品创建订单，自动验证商品有效性，创建成功后从购物车中删除对应商品。
认证要求：需要 Bearer Token
参数名 类型 必填 说明
cart_item_ids Array[Number] 必填 购物车项ID数组
receiver_info Object 必填 收货信息
receiver_info.receiver_name String 必填 收货人姓名
receiver_info.receiver_address String 必填 收货地址
receiver_info.receiver_phone String 可选 联系电话（与receiver_mobile至少填一项）
receiver_info.receiver_mobile String 可选 手机号（与receiver_phone至少填一项）
receiver_info.receiver_state String 可选 省份
receiver_info.receiver_city String 可选 城市
receiver_info.receiver_district String 可选 区县
receiver_info.receiver_email String 可选 邮箱
receiver_info.receiver_zip String 可选 邮编
receiver_info.freight Number 可选 运费（默认0）
buyer_message String 可选 买家留言
shop_id Number 必填 聚水潭店铺编号
请求示例：
Headers: Authorization: Bearer <JWT_TOKEN> Content-Type: application/json Body: { "cart_item_ids": [1, 2, 3], "shop_id": 12345, "receiver_info": { "receiver_name": "张三", "receiver_address": "北京市朝阳区xxx街道xxx号", "receiver_state": "北京", "receiver_city": "北京市", "receiver_district": "朝阳区", "receiver_mobile": "13800138000", "receiver_zip": "100000" }, "buyer_message": "请尽快发货", "freight": 10 }

POST /api/orders/:id/confirm

功能描述：确认收货。买家确认收货后，订单状态变更为 TRADE_FINISHED（交易完成）。
认证要求：需要 Bearer Token
参数 类型 必填 说明
id Number 必填 订单ID（路径参数）
响应示例（成功）：
{ "code": 0, "message": "确认收货成功", "data": { "order_id": 39, "order_no": "202512031102362242", "shop_status": "TRADE_FINISHED", "confirm_time": "2025-12-03T04:44:27.170Z" } }
响应示例（失败 - 状态不允许）：
{ "code": 400, "message": "当前订单状态为\"已完成\"，无法确认收货" }
注意事项：
  • 只有状态为 WAIT_BUYER_CONFIRM_GOODS（待收货）的订单才能确认收货
  • 确认收货后订单状态变更为 TRADE_FINISHED（交易完成）
  • 只能确认当前用户自己的订单
  • 确认收货时间会被记录到 confirm_time 字段

6. 微信支付接口

支付系统说明

  • 支付方式：微信小程序支付（JSAPI）
  • 支付流程：创建支付订单 → 调用微信支付 → 支付回调 → 更新订单状态
  • 支付回调：微信支付成功后会自动回调服务器，更新订单和支付状态，并自动同步订单到聚水潭
  • 状态查询：支持主动查询支付状态，同步微信支付状态到本地
  • 安全验证：所有支付回调都会验证签名，确保数据安全

POST /api/payment/v3/create

功能描述：创建微信支付 v3 订单（专门为前端封装）。强制使用 API v3，返回格式化的支付参数，前端可直接调用微信小程序支付 API。推荐在已启用 API v3 的环境中使用。
认证要求：需要 Bearer Token
API v3 特点：
  • 使用 RSA-SHA256 签名（更安全）
  • 签名类型固定为 "RSA"
  • 返回格式专门为 v3 优化
  • 需要配置 WX_PAY_API_V3=true 和相关证书
参数名 类型 必填 说明
order_id Number 必填 订单ID
请求示例：
Headers: Authorization: Bearer <JWT_TOKEN> Content-Type: application/json Body: { "order_id": 1 }
响应示例：
{ "code": 0, "message": "支付订单创建成功", "data": { "order_id": 1, "order_no": "20250101120000001", "total_fee": 29900, "payment": { "appId": "wx1234567890abcdef", "timeStamp": "1704067200", "nonceStr": "5K8264ILTKCH16CQ2502SI8ZNMTM67VS", "package": "prepay_id=wx1234567890abcdef", "signType": "RSA", "paySign": "abc123def456..." }, "api_version": "v3", "prepay_id": "wx1234567890abcdef" } }
小程序调用支付（v3）：
// 使用返回的payment参数调用wx.requestPayment // 注意：signType 为 "RSA"（不是 "MD5"） wx.requestPayment({ appId: payment.appId, timeStamp: payment.timeStamp, nonceStr: payment.nonceStr, package: payment.package, signType: payment.signType, // "RSA" paySign: payment.paySign, success: function(res) { console.log('支付成功', res); // 跳转到支付成功页面 wx.redirectTo({ url: '/pages/payment/success?order_id=1' }); }, fail: function(err) { console.error('支付失败', err); if (err.errMsg.includes('cancel')) { wx.showToast({ title: '支付已取消', icon: 'none' }); } else { wx.showToast({ title: '支付失败，请重试', icon: 'none' }); } } });
与通用接口的区别：
特性 /api/payment/create /api/payment/v3/create
API版本 自动选择（v2/v3） 强制使用 v3
返回格式 通用格式 专门为v3优化
签名类型 MD5 或 RSA 固定为 RSA
错误提示 通用错误 v3特定错误
适用场景 兼容v2和v3 仅v3环境
错误响应示例：
// API v3 未启用 { "code": 400, "message": "微信支付v3未启用，请检查环境变量 WX_PAY_API_V3=true" } // 订单已支付 { "code": 0, "message": "订单已支付", "data": { "order_id": 1, "payment_id": 1, "paid": true } }

GET /api/payment/status/:order_id

功能描述：查询订单支付状态。如果本地未支付，会主动查询微信支付状态并同步到本地。
认证要求：需要 Bearer Token

POST /api/payment/notify

功能描述：微信支付回调接口。微信支付成功后会自动调用此接口，更新订单和支付状态，并自动同步订单到聚水潭。此接口不需要认证，但会验证签名确保安全。
认证要求：无需认证（微信服务器调用）
支付回调处理流程：
  1. 验证支付回调签名和订单信息
  2. 更新订单状态为 WAIT_SELLER_SEND_GOODS
  3. 记录支付信息到 order_payments 表
  4. 异步同步订单到聚水潭（不阻塞回调响应）
  5. 返回成功响应给微信服务器
注意事项：
  • 此接口由微信服务器调用，不需要前端调用
  • 需要在微信商户平台配置回调地址：{BASE_URL}/api/payment/notify
  • API v2 回调数据为 XML 格式，API v3 回调数据为 JSON 格式
  • 系统会自动识别回调格式并验证签名
  • 回调成功后会自动更新订单状态为"等待卖家发货"
  • 回调成功后会创建支付记录
  • 支付成功后会自动同步订单到聚水潭（异步执行，不阻塞回调响应）
  • 同步失败不影响支付流程，只记录日志，可以后续手动重试

PUT /api/payment/order/:order_id

功能描述：手动更新订单支付信息。适用于线下支付、其他支付方式、手动标记已支付等场景。
认证要求：需要 Bearer Token
参数名 类型 必填 说明
order_id Number 必填 订单ID（路径参数）
outer_pay_id String 必填 外部支付单号，在订单内必须唯一
pay_date String 可选 支付日期，ISO 8601格式（默认当前时间）
payment String 可选 支付方式（默认"其他支付"）
seller_account String 可选 收款账户（默认"默认账户"）
buyer_account String 可选 买家支付账号（默认用户ID）
amount Number 可选 支付金额，单位元（默认订单金额）
update_order_status Boolean 可选 是否更新订单状态（默认true）。如果为true且订单状态为WAIT_BUYER_PAY，会更新为WAIT_SELLER_SEND_GOODS
请求示例：
Headers: Authorization: Bearer <JWT_TOKEN> Content-Type: application/json Body: { "outer_pay_id": "MANUAL_PAY_202512010024313768_1234567890", "pay_date": "2025-12-01T10:30:00.000Z", "payment": "线下支付", "seller_account": "默认账户", "buyer_account": "66", "amount": 2699.00, "update_order_status": true }
响应示例：
{ "code": 0, "message": "订单支付信息更新成功", "data": { "order_id": 15, "order_no": "202512010024313768", "payment": { "outer_pay_id": "MANUAL_PAY_202512010024313768_1234567890", "pay_date": "2025-12-01T10:30:00.000Z", "payment": "线下支付", "amount": 2699.00 }, "order_status_updated": true } }
使用场景：
  • 线下支付：用户通过线下方式支付订单
  • 其他支付方式：使用非微信支付的支付方式
  • 手动标记：管理员手动标记订单为已支付
  • 支付信息修正：修正或补充支付信息
注意事项：
  • 支付单号在订单内必须唯一，如果已存在会更新而不是创建新记录
  • 金额单位为"元"，与订单金额单位一致
  • 如果订单已同步到聚水潭，更新支付信息后会自动重新同步（异步执行）
  • 只能更新当前用户自己的订单
